# W85 - Real Dataset Benchmark Implementation

## Context

The benchmark harness currently supports synthetic datasets only. We need persistent real Hugging Face dataset support so SDK quality can be measured on budget-limited active-learning settings where random selection is weak.

## Goal

Implement real-dataset benchmark support, tests, and documentation.

## Responsibility Boundaries

You own benchmark harness/test/doc files only.

## In Scope

- `benchmarks/sdk_first_benchmark.py`
- `benchmarks/README.md`
- New or existing tests under `tests/` related to benchmark datasets/metrics.
- Optional benchmark result docs only if generated by a smoke run.

## Out of Scope

- SDK runtime/strategy implementation under `src/**`.
- `pyproject.toml` unless a missing dependency is unavoidable.
- Full benchmark reruns over all strategies/seeds unless needed and practical.

## Requirements

Add real dataset registry/support for:

- `clinc_oos_imbalanced`: Hugging Face `clinc/clinc_oos`, config `imbalanced`, train/validation/test, text column `text`, label column `intent`.
- `clinc_oos_plus`: Hugging Face `clinc/clinc_oos`, config `plus`, train/validation/test, text column `text`, label column `intent`.
- `banking77`: Hugging Face `mteb/banking77`, config `default`, train/test, text column `text`, label column `label`.
- `dair_ai_emotion`: Hugging Face `dair-ai/emotion`, config `split`, train/validation/test, text column `text`, label column `label`, marked as sanity/easy coverage.

Document `go_emotions_simplified` as deferred/not part of single-label default because it is multi-label and requires explicit conversion policy.

Required benchmark behavior:

- Real datasets are opt-in through CLI dataset names, not part of existing smoke/full defaults unless explicitly chosen.
- Add a `real_smoke` preset or equivalent that runs a small representative matrix.
- Add budgets suitable for real datasets, e.g. `100,200,300,500,800`.
- Use opaque sample ids and opaque group ids generated from dataset/split/index; do not expose source labels in ids/meta/group ids.
- Preserve private true labels only in benchmark-owned state for training/evaluation/diagnostics.
- Add budget/no-budget metrics:
  - class coverage count/fraction;
  - missing class count/fraction;
  - missing test-support-weighted fraction if practical;
  - macro-F1/weighted-F1/accuracy at each budget;
  - full-supervised no-budget reference row if practical or a separate `full_train_reference` summary.
- Include all SDK strategy names already supported by `strategy_specs()` when requested. The preset may choose a smaller default strategy subset for runtime.
- Keep strict JSON artifacts.

Required tests:

- Dataset registry contains expected real datasets with correct metadata.
- Real dataset loader can be monkeypatched/faked so unit tests do not download from HF.
- Opaque id/group generation does not leak label names or numeric label ids directly.
- Class coverage and missing-class metrics are correct on a toy dataset.
- CLI parsing accepts the new preset/dataset names.
- Real datasets are not included in old `smoke`/`full` defaults by accident.

## Expected Validation

- Run targeted benchmark tests.
- Run full test suite if feasible.
- Run a tiny real smoke with constrained strategies/budget if feasible and not too slow.

## Dependencies

Can use design feedback from W84 if available, but implementation can proceed with the requirements above.
