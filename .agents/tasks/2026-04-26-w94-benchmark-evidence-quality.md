# W94 Benchmark Evidence Quality

## Context
Senior acceptance review added a strict xfail showing the audit benchmark fixture does not distinguish entropy, margin, and least-confidence formulas. It also notes that benchmark runs overwrite accepted evidence in fixed output directories.

## Goal
Make the benchmark fixture discriminating and make artifact writes non-destructive by default.

## Ownership
Primary write scope:
- `benchmarks/audit_sdk_vs_manual.py`
- `tests/test_senior_acceptance_blockers.py` for converting the benchmark fixture xfail into a normal passing test
- benchmark result artifacts only if regenerated by validation commands

## In Scope
- Add adversarial frozen-probability candidates so entropy, margin, and least-confidence produce distinguishable selected orders.
- Preserve SDK/manual formula parity for every supported strategy.
- Make output directory writes fail by default when existing artifacts would be overwritten.
- Add an explicit `--overwrite` CLI switch and internal argument for intentional replacement.

## Out of Scope
- External modAL/scikit-activeml adapters.
- End-to-end active-learning quality claims.
- SDK runtime optimization beyond keeping current benchmark usable.

## Constraints
- Keep benchmark contract and caveats honest.
- Do not silently overwrite existing summary/comparison/analysis artifacts.
- Existing tests may need to use temporary directories or `overwrite=True`.

## Expected Tests
- `tests/test_senior_acceptance_blockers.py::test_manual_benchmark_fixture_must_distinguish_uncertainty_formulas`
- `tests/test_audit_benchmark_comparison.py`
- Direct benchmark smoke command with a fresh directory or `--overwrite`.

## Dependencies
Independent of W94 backend and scheduler/cache tasks.
