# W19 - Fix Benchmark Review Findings

## Relation To Overall Task
R28 reviewed the W18 benchmark harness and found two required fixes before the harness can be used to drive SDK optimization:
1. Acquisition provider leaks true labels via `DataSample.meta`.
2. `summary.json` contains non-standard `NaN` values.

## Assumptions And Resolved Ambiguities
- R28 findings are accepted as valid.
- The benchmark must prevent oracle-label access during acquisition, including for future custom SDK strategies that may inspect `context.get_samples(...)`.
- True labels may remain in benchmark-private data structures for initial seed construction, post-selection oracle labeling, training, evaluation, and diagnostics.
- Machine-readable JSON artifacts must be strict JSON.

## Goal And Expected Result
Fix the benchmark harness so:
- `SelectionContext` and any provider used during acquisition do not expose true labels.
- Missing metrics are serialized as `null` or omitted in JSON, never as `NaN`.
- JSON writing fails fast if non-standard float values remain.
- Smoke artifacts are regenerated and pass strict JSON parsing.

- Do not mutate private SDK state.

## High-Level Execution Plan
- Modify the provider or add a flag so acquisition samples expose text/split/group but not true label.
- Ensure diagnostics/training/evaluation still use benchmark-private `BenchmarkSample.label`.
- Add a JSON-safe sanitizer or custom strict dumps path that converts NaN/inf to `None`.
- Write `manifest.json` and `summary.json` with `allow_nan=False`.
- Regenerate smoke artifacts.
- Run relevant validations.

## Step -> Verify Plan

- Regenerate smoke -> verify metrics/selections row counts and artifact existence.
